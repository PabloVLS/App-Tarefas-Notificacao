========================================================
   COLINHA PARA O VÍDEO - APP "MINHAS TAREFAS"
   (explicação simples de cada método do código)
========================================================

Dica: vá abrindo os arquivos na ordem abaixo enquanto fala.
A frase em "=>" é o que você pode falar no vídeo.


--------------------------------------------------------
1) Tarefa.kt   (pasta model)
--------------------------------------------------------
É o "molde" de uma tarefa. Guarda os dados de cada lembrete.

- data class Tarefa(id, titulo, categoria, mensagem, horario, concluida)
  => "Aqui eu defino o modelo da tarefa: cada tarefa tem um id,
      um título, uma categoria, uma mensagem, o horário do lembrete
      (em milissegundos) e se já foi concluída ou não."


--------------------------------------------------------
2) RepositorioTarefas.kt   (pasta data)
--------------------------------------------------------
Salva e lê os dados no celular usando SharedPreferences (em formato JSON).
Também guarda a preferência de tema (claro/escuro).

- carregarTarefas()
  => "Lê as tarefas salvas no celular e devolve a lista."

- salvarTarefas(lista)
  => "Converte a lista de tarefas em JSON e salva no aparelho."

- proximoId()
  => "Gera um número de id único e sequencial para cada nova tarefa."

- temaEscuro (get/set)
  => "Guarda e recupera se o usuário escolheu tema claro ou escuro,
      pra manter a preferência mesmo fechando o app."


--------------------------------------------------------
3) NotificacaoHelper.kt   (pasta notification)
--------------------------------------------------------
Cuida de tudo relacionado a MOSTRAR a notificação.

- criarCanal(context)
  => "A partir do Android 8, toda notificação precisa de um 'canal'.
      Aqui eu crio esse canal de lembretes."

- temPermissaoNotificacao(context)
  => "Verifica se o usuário deu permissão de notificação (Android 13+)."

- mostrar(context, id, titulo, categoria, mensagem)
  => "Monta a notificação com o título e a mensagem da tarefa e a exibe
      na tela. É usada tanto no horário agendado quanto no botão de teste."


--------------------------------------------------------
4) AgendadorLembrete.kt   (pasta notification)
--------------------------------------------------------
Cuida de AGENDAR o lembrete no horário certo usando o AlarmManager.

- agendar(context, tarefa)
  => "Aqui eu uso o AlarmManager pra programar a notificação no horário
      exato da tarefa, funcionando mesmo com o app fechado."

- cancelar(context, tarefa)
  => "Cancela o alarme de uma tarefa (quando ela é removida ou concluída)."

- podeAgendarExato(context)
  => "Verifica se o sistema permite alarme exato (Android 12+ pede
      a permissão 'Alarmes e lembretes')."

- abrirConfiguracoesAlarmeExato(context)
  => "Abre a tela de configuração pra o usuário liberar o alarme exato."

- criarPendingIntent(context, tarefa)
  => "Cria o 'aviso' (PendingIntent) que o alarme vai disparar na hora certa,
      levando os dados da tarefa para o receptor."


--------------------------------------------------------
5) LembreteReceiver.kt   (pasta notification)
--------------------------------------------------------
É quem "recebe" o disparo do alarme no horário programado.

- onReceive(context, intent)
  => "Quando chega o horário, o AlarmManager chama esse método.
      Ele pega os dados da tarefa e manda mostrar a notificação."


--------------------------------------------------------
6) BootReceiver.kt   (pasta notification)
--------------------------------------------------------
Reagenda os lembretes depois que o celular reinicia.

- onReceive(context, intent)
  => "Quando o celular é ligado/reiniciado, os alarmes se perdem.
      Esse método reagenda todos os lembretes futuros automaticamente."


--------------------------------------------------------
7) TarefaViewModel.kt   (pasta ui)
--------------------------------------------------------
É o "cérebro" da tela: guarda o estado e contém as regras (MVVM).

- adicionarTarefa(titulo, categoria, mensagem, horario)
  => "Cria a nova tarefa, salva na lista e já agenda o lembrete dela."

- alternarConcluida(tarefa)
  => "Marca ou desmarca a tarefa como concluída. Se concluiu, cancela
      o lembrete; se reativou, agenda de novo."

- removerTarefa(tarefa)
  => "Remove a tarefa da lista e cancela o alarme dela."

- testarNotificacao()
  => "Dispara uma notificação na hora, só pra testar se está funcionando."

- definirTema(escuro)
  => "Troca entre tema claro e escuro e salva a escolha do usuário."

- atualizarLista(lista)
  => "Função interna que ordena a lista por horário e salva no repositório."


--------------------------------------------------------
8) TarefasApplication.kt   (raiz)
--------------------------------------------------------
- onCreate()
  => "Roda quando o app inicia. Aqui eu já crio o canal de notificação."


--------------------------------------------------------
9) MainActivity.kt   (raiz)
--------------------------------------------------------
É a tela inicial do app (Activity principal).

- onCreate(savedInstanceState)
  => "Quando o app abre, eu peço a permissão de notificação, verifico a
      permissão de alarme exato e mostro a tela principal já com o tema
      escolhido pelo usuário."

- pedirNotificacao (launcher)
  => "É o pedido de permissão de notificação que aparece pro usuário no
      Android 13 ou superior."


--------------------------------------------------------
10) Theme.kt   (pasta ui/theme)
--------------------------------------------------------
Controla as cores do app (claro x escuro).

- TarefasTheme(temaEscuro, content)
  => "Aplica o conjunto de cores claro ou escuro em todo o app, conforme
      o usuário escolher no interruptor."


--------------------------------------------------------
11) TelaListaTarefas.kt   (pasta ui)  -> a interface (Compose)
--------------------------------------------------------
Desenha a tela: a lista, o botão de adicionar e o formulário.

- TelaListaTarefas(viewModel)
  => "É a tela principal: tem a barra no topo com o interruptor de tema e
      o botão de testar notificação, a lista de tarefas e o botão + pra
      adicionar uma nova."

- EstadoVazio()
  => "Mostra uma mensagem amigável quando ainda não tem nenhuma tarefa."

- ItemTarefa(tarefa, ...)
  => "Desenha cada tarefa na lista: o título, a categoria, o horário,
      o botão de concluir e o botão de excluir."

- DialogoNovaTarefa(...)
  => "É a janelinha de cadastro: campo de título, escolha da categoria,
      mensagem e o seletor de data e hora do lembrete."

- abrirSeletorDataHora(...)
  => "Abre o calendário e depois o relógio pra o usuário escolher a data
      e a hora do lembrete."

- formatarDataHora(horario)
  => "Transforma o horário (em milissegundos) num texto bonito, tipo
      '25/06/2026 às 14:30'."


========================================================
   RESUMO RÁPIDO PRA FALAR NO INÍCIO DO VÍDEO
========================================================
"Esse é um app de lista de tarefas feito em Kotlin com Jetpack Compose.
Ele usa o padrão MVVM: a TelaListaTarefas é a interface, o TarefaViewModel
guarda o estado e as regras, o RepositorioTarefas salva os dados no celular,
e a parte de notificação usa o AlarmManager pra disparar lembretes no horário
certo, mesmo com o app fechado. Também tem alternância entre tema claro e
escuro, com a preferência salva."
========================================================
